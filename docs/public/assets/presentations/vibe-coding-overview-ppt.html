03 / 07

Key Section

Vibe Coding 是什么

  • 你可以把 Vibe Coding 理解成一种“让 AI 深度参与开发工作流”的方式。
  • 它不只是：
  • 问几个编程问题
  • 让 AI 补全几行代码
  • 复制一段示例就结束
04 / 07

Key Section

为什么不要执着于“找一个最强工具”

  • 这是很多新手一开始最容易掉进去的坑。
  • 大家常问：
  • Trae、Codex、Claude 哪个更强？
  • 我是不是只要学会一个就够了？
  • 有没有一个工具能包办全部流程？
05 / 07

Key Section

Trae：更适合建立项目上下文

  • 如果你刚接手一个仓库，最痛苦的事情通常不是“不会写代码”，而是：
  • 不知道入口在哪
  • 不知道模块怎么分
  • 不知道页面和接口怎么对应
  • 不知道某个功能改动会影响哪里
06 / 07

Key Section

Codex：更适合围绕任务持续推进

  • 当你已经知道要改什么、在哪改、怎么验证时，就进入了更偏工程执行的阶段。
  • Codex 更适合：
  • 读取仓库规范和上下文
  • 修改真实文件
  • 按任务推进实现
