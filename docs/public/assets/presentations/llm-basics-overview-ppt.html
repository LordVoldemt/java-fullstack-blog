Key Section

什么是大模型，为什么它和传统程序不一样

  • 大模型，通常指的是以大规模参数和训练语料为基础构建的语言模型。它最直接的能力，是根据上下文生成符合语言规律的输出。
  • 看起来它像“更聪明的聊天机器人”，但真正关键的区别在于：它不是靠固定规则覆盖所有输入，而是在概率空间里生成最可能的响应。
  • 这意味着，大模型和传统程序的工作方式有本质差异。
  • 传统程序通常是：
  • 规则明确

Prompt：为什么“怎么问”会直接影响结果质量

  • Prompt 可以简单理解成你给模型的输入指令和上下文。很多人第一次上手大模型时，会觉得 Prompt 不就是“提问方式”吗？表面上确实如此，但从工程视角看，它远不只是提...
  • Prompt 至少承担三类职责：
  • 描述任务目标
  • 提供上下文边界
  • 约束输出格式和风格

Key Section

Embedding：为什么大模型应用离不开“向量化理解”

  • 当你开始做知识库问答、语义搜索、相关推荐时，Embedding 几乎是绕不过去的概念。
  • 它的核心思想，是把文本映射成一个高维向量，让机器可以在“语义空间”里比较相似性。传统搜索更多依赖关键词匹配，而 Embedding 让系统能够在一定程度上理解“意思接近但...
  • 比如：
  • “Java 并发” 和 “多线程编程”
  • “缓存雪崩” 和 “大量缓存同时失效”
