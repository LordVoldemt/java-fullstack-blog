# Design - 博客站点首期建设

## 基本信息

- 项目名称：java全栈
- 变更名称：blog-foundation
- 日期：2026-05-02
- 状态：草稿
- 关联 PRD：`.adpp/changes/2026-05-02-blog-foundation/prd.md`

## 1. 设计目标

- 为博客建立稳定、可扩展的信息架构
- 在不引入后端的前提下，完成内容发布、导航组织和阅读体验设计
- 让博客既能承载 Java 后端知识，也能承载 AI、大模型、Vibe Coding 等新方向
- 保持首期实现简单，避免过度主题定制

## 2. 总体方案

本项目采用 `VitePress + Markdown + GitHub Pages` 方案。

### 方案原则

- 内容优先：文章是核心资产，主题只是承载层
- 结构优先：先做好栏目与目录，不先堆复杂首页效果
- 配置优先：能通过 VitePress 配置解决的，不额外造组件
- 定制克制：首期只做必要的首页与导航定制

### 交付形态

- 一个可本地构建的静态博客站点
- 一套稳定的 `docs/` 内容目录结构
- 一套顶部导航与栏目组织方案
- 一个适合长期扩展的首页、栏目页、文章页基础体验

- 文章归属一个主栏目即可，避免跨多个栏目造成维护混乱
- 新增方向优先扩展栏目内容层，尽量不频繁改主导航

## 4. 页面结构设计

### 4.1 首页

#### 页面目标

- 让第一次进入的用户快速知道“这是一个讲什么的博客”
- 展示博客的核心方向：Java、微服务、AI、Vibe Coding、项目实战
- 作为站点入口页，为栏目和代表性文章导流

#### 页面模块

- Hero 区
- 核心方向入口区
- 精选文章区
- 最新文章区
- 关于我 / 关于站点区

#### Hero 区建议文案方向

- 主标题：突出“Java 全栈 + AI 实战”
- 副标题：强调知识分享、项目实战、开发提效
- CTA：进入文章、查看专题、阅读首篇推荐

#### 首页设计原则

- 不做过重动画
- 视觉上要有技术感，但阅读效率优先
- 首屏要清楚表达博客定位，而不是只展示装饰

### 4.2 主栏目页

每个主导航对应一个专题页。

#### 专题页目标

- 解释这个方向包含什么内容
- 展示该方向下的二级栏目
- 给出推荐阅读和学习路径

#### 专题页结构

- 专题简介
- 子栏目列表
- 推荐文章列表
- 学习建议或阅读顺序

### 4.3 文章页

#### 文章页目标

- 适合长文本技术阅读
- 保持正文清晰、目录稳定、代码块易读
- 支持读者快速跳转和复用代码片段

#### 文章页要求

- 自动目录
- 代码高亮
- 标准引用、列表、表格、图片展示
- 上下篇或返回栏目入口的辅助导航可作为后续增强

### 目录设计原则

- 每个主栏目一个 `index.md` 作为专题入口页
- 二级栏目使用语义化英文目录，避免后续频繁改名
- 文章文件名统一使用 `kebab-case`
- 首期不强依赖 tags 和复杂自动归档能力

## 6. VitePress 配置方案

### 配置重点

- `title`、`description`、`lang`
- `themeConfig.nav`
- `themeConfig.sidebar`
- `themeConfig.socialLinks`
- `lastUpdated`
- SEO 基础信息

### 导航策略

- 顶部导航只放主栏目
- 侧边栏按当前主栏目展示二级栏目与文章组
- 首页与专题页通过固定入口增强引导

### 主题扩展策略

- 首期优先使用默认主题能力
- 仅当首页需要差异化展示时，在 `.vitepress/theme/` 中增加少量组件
- 若后续需要文章卡片、专题推荐块，再做组件化扩展

## 7. 内容生产工作流

### 新增文章流程

1. 确定文章所属主栏目与二级栏目
2. 在对应目录下创建 Markdown 文件
3. 补充 frontmatter
4. 更新必要的侧边栏或推荐入口
5. 运行 `npm run docs:build`

### 新增专题流程

1. 先判断是否属于已有主栏目
2. 若是已有主栏目的子方向，只新增二级目录与专题页
3. 若需要新主栏目，应先更新 PRD / design 再调整导航

## 8. 页面与配置边界

### 本期实现边界

- 支持基础首页
- 支持主栏目专题页
- 支持标准文章页
- 支持 GitHub Pages 部署，仓库地址：https://github.com/LordVoldemt/java-fullstack-blog.git

### 暂不实现

- 评论系统
- Algolia 或服务端搜索
- 自动标签页、归档页、作者系统
- 多作者内容管理
- 重度首页动画和复杂数据联动

## 9. 风险与回滚方案

### 风险

- 主栏目过多会造成首期内容稀疏
- 首页模块过多会削弱博客型站点的简洁性
- 主题扩展过早会抬高维护成本

### 缓解策略

- 首期允许部分栏目先只有专题入口，不强求每栏立即有大量文章
- 首页默认只保留 4 到 5 个核心模块
- 能用文档页解决的，不先做组件开发

### 回滚方案

- 如首页定制效果不理想，可回退为默认 VitePress 首页结构
- 如主栏目命名不合适，可在正式发布前只调整专题页与导航文案，不改底层技术方案
- 如目录拆分过细，可合并二级目录，不影响整体主架构
